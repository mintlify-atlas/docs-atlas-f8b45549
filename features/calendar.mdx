# Release Calendar

The release calendar helps users stay up-to-date with new episodes from shows they're watching or have on their watchlist.

## Overview

The release calendar shows:
- Tonight's episodes
- Upcoming episodes (next 7 days)
- Season premieres
- Returning seasons
- Season and series finales

## Release Events

Episode release information is stored in the `releaseEvents` table.

### Release Event Structure

```typescript
releaseEvents: {
  showId: Id<"shows">,
  airDate: string, // "2024-03-15"
  airDateTs: number, // Unix timestamp
  seasonNumber: number,
  episodeNumber: number,
  episodeTitle?: string,
  isPremiere: boolean,
  isReturningSeason: boolean,
  isSeasonFinale: boolean,
  isSeriesFinale: boolean
}
```

### Event Flags

Events are automatically tagged based on episode metadata:

<Accordion title="Event Flag Logic">
```typescript
function getReleaseEventFlags({
  seasonNumber,
  episodeNumber,
  lastEpisodeNumber,
  maxSeasonNumber,
  showStatus
}) {
  return {
    isPremiere: seasonNumber === 1 && episodeNumber === 1,
    isReturningSeason: seasonNumber > 1 && episodeNumber === 1,
    isSeasonFinale: episodeNumber === lastEpisodeNumber,
    isSeriesFinale: 
      showStatus === "Ended" && 
      seasonNumber === maxSeasonNumber && 
      episodeNumber === lastEpisodeNumber
  };
}
```
</Accordion>

## Sync State Management

Release data is cached and refreshed on a schedule.

### Show Release Sync State

```typescript
showReleaseSyncState: {
  showId: Id<"shows">,
  syncedAt?: number,
  expiresAt?: number,
  status: "idle" | "scheduled" | "running" | "ready" | "failed",
  lastError?: string,
  updatedAt: number
}
```

### Sync States

<Steps>
  <Step title="Idle">
    No sync has occurred yet
  </Step>
  
  <Step title="Scheduled">
    Sync has been queued but not started
  </Step>
  
  <Step title="Running">
    Currently fetching and processing episode data
  </Step>
  
  <Step title="Ready">
    Fresh data available, not expired
  </Step>
  
  <Step title="Failed">
    Sync encountered an error
  </Step>
</Steps>

### Staleness Detection

```typescript
function isReleaseSyncStateStale(syncState, now) {
  if (!syncState) return true;
  if (syncState.status === "failed") return true;
  if (syncState.status === "ready" && syncState.expiresAt) {
    return syncState.expiresAt <= now;
  }
  return syncState.status !== "ready";
}
```

<Info>
Release data expires after 12 hours to ensure calendar shows up-to-date information.
</Info>

## Background Refresh

A cron job automatically refreshes stale show data.

### Refresh Process

<Steps>
  <Step title="Find Stale Shows">
    Query shows with expired or failed sync states
  </Step>
  
  <Step title="Claim Shows">
    Mark shows as "scheduled" to prevent duplicate processing
  </Step>
  
  <Step title="Batch Refresh">
    Process shows in batches of 6 to avoid rate limits
  </Step>
  
  <Step title="Fetch Episode Data">
    Query TMDB for season and episode details
  </Step>
  
  <Step title="Build Events">
    Create release event records for future episodes
  </Step>
  
  <Step title="Update State">
    Mark sync as "ready" and set expiration
  </Step>
</Steps>

### Manual Refresh

Users can manually trigger a refresh:

```typescript
const refreshForMe = useAction(api.releaseCalendar.refreshForMe);

const result = await refreshForMe();
// Returns: { scheduledCount: number }
```

## Tracked Shows

Only relevant shows appear in the calendar.

### Tracked Status Types

```typescript
function isTrackedReleaseStatus(status) {
  return status === "watching" || status === "watchlist";
}
```

<Note>
Shows marked as "completed" or "dropped" don't appear in the release calendar.
</Note>

### Loading Tracked Shows

```typescript
async function loadTrackedReleaseShows(ctx, userId, now) {
  // Get user's watching/watchlist shows
  const trackedShowIds = await getTrackedShowIdsForUser(ctx, userId);
  
  // Load show details, sync state, and release events
  const shows = await Promise.all(
    trackedShowIds.map(async (showId) => {
      const show = await ctx.db.get(showId);
      const syncState = await getSyncStateForShow(ctx, showId);
      const events = await ctx.db
        .query("releaseEvents")
        .withIndex("by_show_airDateTs", (q) => 
          q.eq("showId", showId)
        )
        .collect();
      
      return { show, syncState, events };
    })
  );
  
  return shows;
}
```

### Setting Provider Filter

```typescript
const setProviderFilter = useMutation(
  api.releaseCalendar.setProviderFilter
);

await setProviderFilter({
  selectedProviders: ["Netflix", "HBO Max", "Disney+"]
});
```

### Provider Normalization

```typescript
const RELEASE_CALENDAR_PROVIDER_OPTIONS = [
  "Netflix",
  "Disney+",
  "HBO Max",
  "Amazon Prime Video",
  "Apple TV+",
  "Hulu",
  "Paramount+",
  "Peacock"
];

function normalizeSelectedProviders(providers) {
  if (!Array.isArray(providers)) return [];
  return providers.filter((p) => 
    RELEASE_CALENDAR_PROVIDER_OPTIONS.includes(p)
  );
}
```

## Episode Caching

Episode data is cached from TMDB for performance.

### Season Cache

```typescript
// Cached as tmdbDetailsCache with special externalSource
tmdbDetailsCache: {
  externalSource: "tmdb-season",
  externalId: "{showId}:season:{seasonNumber}",
  payload: {
    episodes: [{
      episodeNumber: number,
      airDate: string,
      name: string
    }]
  },
  fetchedAt: number,
  expiresAt: number
}
```

### Episode Cache Jobs

Background job to cache season data:

```typescript
tmdbEpisodeCacheJobs: {
  kind: "season_cache",
  status: "queued" | "running" | "completed" | "failed",
  targetShowCount: number,
  batchSize: number,
  nextOffset: number,
  processedShowCount: number,
  cachedSeasonCount: number,
  skippedSeasonCount: number,
  failedShowCount: number,
  totalShowCount?: number
}
```

<Info>
Caching season data in advance makes calendar loading nearly instantaneous.
</Info>
