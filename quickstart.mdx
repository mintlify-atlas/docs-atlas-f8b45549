This guide will walk you through setting up Plotlist on your local machine and running it for the first time.

## Prerequisites

Before you begin, make sure you have:

- **Node.js** (v18 or later)
- **npm** (comes with Node.js)
- An **Expo-compatible environment**:
  - For iOS: macOS with Xcode
  - For Android: Android Studio
  - For Web: Any modern browser
- A **Convex account** (sign up at [convex.dev](https://convex.dev))

## Quick Setup

<Steps>
  <Step title="Clone the repository">
    Clone Plotlist to your local machine:
    
    ```bash
    git clone https://github.com/EthanMcKanna/Plotlist.git
    cd Plotlist
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages:
    
    ```bash
    npm install
    ```
    
    This installs Expo, React Native, Convex, and all other dependencies.
  </Step>

  <Step title="Set up environment variables">
    Copy the example environment file:
    
    ```bash
    cp .env.example .env
    ```
    
    Edit `.env` and add your credentials. At minimum, you'll need:
    
    ```bash .env
    EXPO_PUBLIC_CONVEX_URL=https://your-project.convex.cloud
    TMDB_API_KEY=your_tmdb_api_key
    TWILIO_ACCOUNT_SID=your_twilio_sid
    TWILIO_AUTH_TOKEN=your_twilio_token
    TWILIO_VERIFY_SERVICE_SID=your_verify_service_sid
    CONTACT_HASH_SECRET=your_random_secret_key
    ```
    
    <Accordion title="How to get API keys">
      **TMDB API Key:**
      1. Create an account at [themoviedb.org](https://www.themoviedb.org/)
      2. Go to Settings → API
      3. Request an API key (free for non-commercial use)
      
      **Twilio Credentials:**
      1. Sign up at [twilio.com](https://www.twilio.com/)
      2. Create a Verify service in the Console
      3. Copy your Account SID, Auth Token, and Verify Service SID
      
      **Contact Hash Secret:**
      Generate a random string (e.g., `openssl rand -hex 32`)
    </Accordion>
    
    See the [Environment Setup](/guides/environment-setup) guide for detailed instructions on obtaining all API keys.
  </Step>

  <Step title="Start the Convex development server">
    Start Convex to generate typed client APIs:
    
    ```bash
    npx convex dev
    ```
    
    On first run, you'll need to:
    - Log in to your Convex account
    - Create a new project or select an existing one
    - Link this directory to your Convex deployment
    
    <Note>
      Keep this terminal running. Convex watches for schema changes and regenerates TypeScript types automatically.
    </Note>
  </Step>

  <Step title="Run the app">
    In a new terminal, start the Expo development server:
    
    ```bash
    npm start
    ```
    
    You'll see a QR code and menu with options:
    - Press **`i`** to open iOS simulator
    - Press **`a`** to open Android emulator  
    - Press **`w`** to open in web browser
    - Scan the QR code with Expo Go app on your phone
  </Step>
</Steps>

## Common Issues

### "Cannot find module 'convex/_generated'"

This means Convex hasn't generated the typed client yet. Make sure you've run:

```bash
npx convex dev
```

### "EXPO_PUBLIC_CONVEX_URL is not defined"

Check that your `.env` file has the Convex URL set:

```bash
EXPO_PUBLIC_CONVEX_URL=https://your-project.convex.cloud
```

Restart the Expo server after updating `.env` files.

### Expo stuck on "Waiting for connection"

- Ensure your device/simulator and computer are on the same network
- Try clearing cache: press `Shift+r` in the Expo terminal
- Restart Expo: `npm start -- --clear`

## Development Workflow

### Two Terminals

You need **two terminals running simultaneously**:

**Terminal 1 - Convex backend:**
```bash
npx convex dev
```

**Terminal 2 - Expo frontend:**
```bash
npm start
```

### Hot Reload

Expo supports fast refresh:
- Save a file and see changes instantly
- Press **`r`** to manually reload
- Press **`Shift+r`** to reload and clear cache

### Quality Checks

Before committing code, run:

```bash
npm run check
```

This runs TypeScript type checking, ESLint, and all tests.
