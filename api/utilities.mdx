## Authentication Storage

### authStorage

Cross-platform secure token storage for Convex Auth.

```ts lib/authStorage.ts
import { authStorage } from "../lib/authStorage";

// Platform-adaptive storage
// - Web: localStorage
// - Native: expo-secure-store with in-memory fallback

const storage: TokenStorage = {
  getItem: (key: string) => Promise<string | null>;
  setItem: (key: string, value: string) => Promise<void>;
  removeItem: (key: string) => Promise<void>;
};
```

**Key Features:**
- Automatic platform detection (web vs native)
- Secure keychain storage on iOS/Android
- Graceful fallback to in-memory storage
- Key sanitization (alphanumeric, dots, dashes, underscores only)

### clearAuthTokens

Clears all authentication tokens to force re-login.

```ts
export async function clearAuthTokens(): Promise<void>
```

**Usage:**
```ts
import { clearAuthTokens } from "../lib/authStorage";

await clearAuthTokens();
// User will need to sign in again
```

## Phone Number Utilities

### formatPhoneNumber

Formats US phone numbers as (XXX) XXX-XXXX while typing.

```ts lib/phone.ts
export function formatPhoneNumber(value: string): string
```

**Usage:**
```ts
import { formatPhoneNumber } from "../lib/phone";

const formatted = formatPhoneNumber("5551234567");
// Returns: "(555) 123-4567"

const partial = formatPhoneNumber("555");
// Returns: "555"
```

### normalizePhoneNumber

Converts phone numbers to E.164 format (+1XXXXXXXXXX).

```ts
export function normalizePhoneNumber(value: string): string | null
```

**Features:**
- Handles US 10-digit numbers
- Handles numbers with country code
- Returns null for invalid formats
- Validates length (8-15 digits)

**Usage:**
```ts
import { normalizePhoneNumber } from "../lib/phone";

normalizePhoneNumber("(555) 123-4567");
// Returns: "+15551234567"

normalizePhoneNumber("+447700900123");
// Returns: "+447700900123"

normalizePhoneNumber("invalid");
// Returns: null
```

### buildReleaseCalendarData

Builds paginated, filtered calendar data.

```ts
export function buildReleaseCalendarData(args: {
  shows: ReleaseCalendarShowSource[];
  today: string;
  view: ReleaseCalendarView;
  selectedProviders?: string[] | null;
  limit?: number;
  cursor?: string | null;
}): {
  groups: ReleaseCalendarGroup[];
  selectedProviders: string[];
  totalItems: number;
  staleShowIds: string[];
  continueCursor: string;
  isDone: boolean;
}
```

**Usage:**
```ts
import { buildReleaseCalendarData } from "../lib/releaseCalendar";

const data = buildReleaseCalendarData({
  shows: trackedShows,
  today: "2024-03-15",
  view: "tonight",
  selectedProviders: ["Netflix", "Max"],
  limit: 25,
  cursor: undefined,
});

// Render groups
data.groups.forEach(group => {
  console.log(group.airDate); // "2024-03-15"
  group.items.forEach(item => {
    console.log(item.show.title, `S${item.seasonNumber}E${item.episodeNumber}`);
  });
});

// Load next page
if (!data.isDone) {
  const nextPage = buildReleaseCalendarData({
    ...args,
    cursor: data.continueCursor,
  });
}
```

## Best Practices

### Error Handling

All async utilities may throw errors - wrap in try/catch:

```ts
try {
  const contacts = await loadDeviceContacts();
} catch (error) {
  console.error("Failed to load contacts:", error);
  // Show user-friendly message
}
```

### Platform Detection

Some utilities behave differently on web vs native:

```ts
import { Platform } from "react-native";

if (Platform.OS === "web") {
  // Web-specific behavior
} else {
  // Native-specific behavior
}
```

### Date Handling

All date utilities use `date-fns` for consistent formatting:

```ts
import { formatDate } from "../lib/format";
import { format } from "date-fns";

// Use utility functions when available
const formatted = formatDate(timestamp);

// Use date-fns directly for custom formats
const custom = format(new Date(timestamp), "yyyy-MM-dd");
```
